# 个人 IP 定位卡

## 基础定位

**身份标签**：资深后端工程师 / 技术管理者 / 3岁孩子奶爸

**一句话定位**：
> 帮助35岁左右、长期停留在执行层的技术人，提升判断力、表达力和成长结构，把多年经验转化成内容与资产，找到职场之外的第二增长曲线。

**公开身份线**（用于文章署名/简介）：
> 10年程序员，专注技术判断力与职场第二曲线

---

## 目标受众（Audience Ladder）

### 核心受众（Primary）
- **年龄**：32-38岁
- **职业**：程序员/工程师，5-10年经验
- **状态**：长期停留在执行层，重复性工作占比高
- **痛点**：
  - 技术判断力不够，只能执行别人决策
  - 表达能力弱，想法说不清楚
  - 担心35岁危机，但没有突破方向
  - 有多年经验，但不知道怎么转化为资产
- **渴望**：建立不可替代性，找到第二增长曲线

### 扩展受众（Secondary）
- 想从技术转管理的人
- 想做副业/内容创作的技术人
- 对职业规划迷茫的中层技术人

### 避免受众（Avoid）
- 刚入行的初级程序员（还没到瓶颈期，无法共鸣）
- 纯求技术干货的读者（不是技术教程账号）
- 只想躺平的人（与成长导向不符）

## 内容支柱（Content Pillars）

### 支柱 1：技术判断力（核心支柱）
**主题**：
- 如何从技术执行者变成技术决策者
- 复杂系统的判断框架
- 技术选型的思考维度
- 从"写代码"到"做判断"的转变

**证明什么**：这个人有深度技术思考，不只是代码工人

**占比**：40%

### 支柱 2：表达力与内容资产（增长支柱）
**主题**：
- 技术人的写作与表达训练
- 如何把工作经验转化为内容
- 建立个人品牌与内容资产
- 技术写作的方法与技巧

**证明什么**：经验可以转化为复利资产，不只是时间换钱

**占比**：30%

### 支柱 3：职场第二曲线（变现支柱）
**主题**：
- 35岁+程序员的职业转型路径
- 副业探索与一人公司
- 从技术到商业的思维转变
- 职场之外的增长可能性

**证明什么**：有实战经验的中年职场人，不只是理论派

**占比**：20%

### 支柱 4：成长结构与心态（连接支柱）
**主题**：
- 执行层到决策层的成长路径
- 职场倦怠与动力重建
- 工作与家庭的平衡（奶爸视角）
- 长期主义与耐心积累

**证明什么**：真实的中年职场人，有人味，不是成功学导师

**占比**：10%

## 声音与风格（Voice Profile）

**基调**：
- 冷静、克制、低焦虑
- 不说"你可以的"，说"这样做可能有效"
- 承认困难，但提供结构化的出路

**语言习惯**：
- 短句为主，克制修饰
- 用具体场景，不用抽象概念
- 适当使用技术类比（系统、架构、复利）
- 第一人称分享，但不自恋

**禁忌**：
- ❌ 励志鸡汤（"只要努力就能成功"）
- ❌ 贩卖焦虑（"35岁必死"）
- ❌ 纯理论说教（没有实战经验支撑）
- ❌ 过度个人化（变成日记流水账）

| 我们是 | 我们不是 |
|--------|---------|
| 技术判断力训练 | 技术教程或代码分享 |
| 实战派的中年职场人 | 年轻热血的职场导师 |
| 把经验转化为资产 | 快速致富或副业暴富 |
| 结构化成长路径 | 空洞的自我提升 |
| 奶爸视角的真实平衡 | 工作狂或躺平派 |

## 话题筛选标准

**通过标准**（必须满足2项以上）：
- [ ] 切中35岁执行层技术人的真实痛点
- [ ] 与"判断力/表达力/第二曲线"相关
- [ ] 有真实经验支撑，不是纯理论
- [ ] 能够提供结构化方法，不只是情绪共鸣
- [ ] 有差异化角度，不是人云亦云

**拒绝标准**（满足1项即拒绝）：
- [ ] 纯技术干货（代码/框架/工具）
- [ ] 与目标受众年龄/阶段不符
- [ ] 鸡汤励志，缺乏实操
- [ ] 热点追逐，与定位无关

## 使用说明

**在内容创作工作流中**：
1. 话题发现阶段：用此卡筛选话题是否匹配
2. 写作阶段：用此卡检查语气、角度、深度
3. 发布前：用此卡确认内容角色和预期效果
4. 复盘阶段：用此卡评估是否强化IP识别度

**更新频率**：
- 每月回顾一次，检查是否有定位漂移
- 根据反馈调整受众定义和内容占比
- 随着阶段变化，更新30天目标
