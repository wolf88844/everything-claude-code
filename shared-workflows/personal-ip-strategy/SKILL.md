# Personal IP Strategy

Do not treat a personal IP as a pile of posts.
Treat it as a long-term positioning system that content workflows must serve.

## When to Use

Use this skill when the user wants to:
- clarify or recalibrate personal IP positioning
- identify the right audience and avoid content drift
- define long-term content pillars
- map content roles such as growth, authority, bridge, and conversion
- decide what matters most at the current stage of the account
- understand why content may be executing well but the IP still feels blurry
- build the strategic layer above topic selection and article execution

## Core Principle

Execution quality is not enough.
A personal IP becomes strong when repeated content starts making the same person, same judgment style, and same thematic territory easier to recognize.

This skill sits above:
- topic workflows
- article workflows
- cross-platform workflows
- visual packaging workflows

It answers:
- who this IP is for
- what territory it should occupy
- what kind of recognition it should build
- what each content type is supposed to do
- what stage-specific priorities matter now

## External Strategy Roots

This skill borrows from four external strategy patterns:
1. positioning frameworks -> sharpen the occupied territory
2. creator voice and messaging frameworks -> improve public recognizability
3. content pillar strategy -> reduce theme drift and increase repeatable recognition
4. creator editorial systems -> connect one-off articles to a compounding asset base

Do not use these frameworks mechanically.
Always adapt them to the user's real content, audience, and publishing stage.

## Default Modules

### Module 1: Positioning Calibration

Clarify:
1. who the user helps
2. what painful or costly pattern they explain better than others
3. what differentiates this IP from adjacent accounts
4. what this IP is not
5. what one-sentence public-facing identity the account should reinforce

Deliver:
- long-form positioning statement
- short public-facing identity line
- "we are / we are not" boundary
- positioning risks or blur points

### Module 2: Audience Ladder

Split the audience into three layers:
1. core audience -> the people worth deeply serving first
2. expansion audience -> the people who may enter through broader recognition content
3. pollution audience -> the people likely to dilute the account if over-served

Deliver:
- core audience definition
- expansion audience definition
- pollution risks
- what kind of content each layer responds to

### Module 3: Content Pillar Architecture

Define the long-term content pillars.
Aim for 3 to 5 max.
Each pillar should be specific enough to build recognition and broad enough to sustain repeated publishing.

Also define:
- supporting or secondary pillars
- themes to avoid or deprioritize
- what each pillar proves about the IP

Deliver:
- primary pillars
- secondary pillars
- low-priority or avoid list
- why each pillar matters strategically

### Module 4: Content Role Map

Not every piece of content should do the same job.
Map content into roles such as:
- growth / discovery
- recognition / identity reinforcement
- authority / diagnosis
- bridge / transition into the deeper system
- conversion / trust-building

Deliver:
- role definitions
- examples of what fits each role
- current recommended role mix
- common role imbalance to watch

### Module 5: Stage Strategy

Diagnose the account's current stage.
Typical stages include:
- early formation
- recognition building
- small-scale validation
- system tightening
- expansion and amplification

Then state:
- what matters most at this stage
- what not to over-optimize yet
- what success should look like in the next 30 days

Deliver:
- current stage
- 3 stage priorities
- 3 things not to chase yet
- next-phase indicators

### Module 6: IP Drift Check

Check whether current content is strengthening the same recognizable identity or drifting outward.
Look for:
- too many broad topics with weak tie-back
- strong single articles but weak overall recognizability
- pillar imbalance
- platform drift
- tone drift

Deliver:
- current drift level
- main drift source
- what to tighten immediately



### Module 7: Success Engineering

Do not stop at deciding what to publish.
Define what each topic is supposed to win.

For each planned topic, specify:
- primary goal: recognition, spread, relationship, or conversion
- expected level: S, A, B, or C
- why this topic is naturally suited to that goal
- minimum success signal
- most likely failure mode

This module should constrain drafting, not only planning.
If a topic's success condition is unclear, the article workflow should slow down before drafting.

Deliver:
- topic-level success map
- topic suitability judgment
- expected outcome per content role
- stage-appropriate win conditions

## Recommended Output Format

Default output should include:
1. current IP diagnosis
2. positioning calibration
3. audience ladder
4. content pillar architecture
5. content role map
6. stage strategy
7. IP drift check
8. success engineering map
9. 30-day strategic recommendations

## Practical Rule

Do not let this strategy layer become vague branding language.
Every strategic claim should be testable through:
- titles chosen
- topics selected
- article patterns repeated
- what the audience starts saying about the account

## Integration Rule

This skill should guide and constrain:
- `social-inspiration-workflow`
- `wechat-growth-workflow`
- `xiaohongshu-content`
- `article-writing`

If strategy and execution conflict, fix the strategy or the content mix.
Do not simply keep producing more content.
